# ストレージ

Blob・Queue・Event Grid によるイベント駆動と、コンテナからの閉域アクセスの方針をまとめる。詳細なブループリントは `infra/blueprint.md` の「ストレージ・イベント駆動」を参照する。

## リソースと役割

| リソース                   | 役割                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Storage Account**        | Blob と **Queue** を提供するアカウント（同一アカウントでまとめる想定）。Private Endpoint。**冗長性（LRS / ZRS / GRS 等）はアカウントごとに選定**。`minimum_tls_version` は **TLS 1.2 以上**を明示。                                                                                                                                                                                                                                                                                                          |
| **Queue Storage**          | Event Grid からメッセージを受け取るキュー（例: ファイル受信用、通知用）。バッチの入力待ち行列。                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Event Grid**             | ストレージの **Blob 作成（等）をサブスクライブ**し、**Queue にメッセージをエンキューする**（システムトピックとイベントサブスクリプション）。**デッドレター先**に Blob コンテナ（例: `eventgrid-deadletter`）を設定する。`publicNetworkAccess = Disabled` 下で Queue へ届けるには、ストレージのネットワークで**「信頼された Azure サービスからのアクセスを許可」**が必要。Blob→`file-reception` 用と Blob→Snowpipe 通知キュー用では**イベントサブスクリプションを別**とし、**エンキュー先キューも別**とする。 |
| **Container App（Batch）** | **キューをコンシューム**し、同一ストレージの **Blob も Private Endpoint 経由で読み書き**する（スケールは KEDA でキュー深度に連動）。                                                                                                                                                                                                                                                                                                                                                                         |

## Blob コンテナ別の保護設定

| コンテナ     | 用途                         | ソフトデリート | バージョニング | 備考                                                                                                                                                                              |
| ------------ | ---------------------------- | -------------- | -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `inbound`    | 外部からの受信ファイル用     | 有効（14日）   | 有効           | Snowflake からの書き込みは **Private Link** 経由。                                                                                                                                |
| `processing` | バッチ処理中のファイル置き場 | 有効（14日）   | 有効           | 長時間滞留はアラート対象。                                                                                                                                                        |
| `processed`  | バッチ処理済みファイル格納   | 有効（7日）    | 無効           |                                                                                                                                                                                   |
| `outbound`   | 外部への送信ファイル用       | 有効（7日）    | 無効           | **当システムが置いた Blob を Snowflake が Private Link 経由で読み取る**領域。bulk load と **Snowpipe** を想定。**ライフサイクル管理**（アーカイブ層・削除ポリシー等）を別途定義。 |

## Queue 別の設定

| キュー名          | 用途                                      | プロデューサー                           | コンシューマー                                                                            | 備考                                                      |
| ----------------- | ----------------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| `file-reception`  | Blob 作成イベントを受けてバッチ起動       | Event Grid                               | Container Apps（Batch）                                                                   | TTL 7日、試行上限5回。超過で `file-reception-poison` へ。 |
| `snowpipe-notify` | Snowpipe 自動インジェストの Blob 到着通知 | Event Grid（**専用サブスクリプション**） | Snowflake（[公式手順](https://docs.snowflake.com/en/user-guide/data-load-azure-private)） | キュー名・DLQ・TTL は連携設計で確定。                     |
| `notification`    | アプリ内通知（メール送信等）              | API / Batch                              | Container Apps（Batch）                                                                   | 試行上限5回、超過で `notification-poison`。               |

## Queue 失敗系の運用方針

- Azure Queue Storage に**ポイズンキューへの自動移送はない**。コンシューマーが `dequeueCount` を確認し、閾値超過時にポイズンキューへ移送するロジックを**アプリ側で実装**する。
- ポイズンキュー（`*-poison`）にメッセージが入った時点でアラート（[監視](./monitor.md)）。
- **visibility timeout** は処理時間の最大値に合わせる。**メッセージに業務キー（冪等性キー）**を含め、重複処理に耐える設計とする。

## インフラ上の考慮（ファイル着信バッチ）

- バッチは **Private Endpoint** 経由で、**同一ストレージアカウント**の Blob と Queue にアクセスする。Container Apps（Batch）の **Managed Identity** に Blob／Queue 用の RBAC（例: Storage Blob Data Contributor、Storage Queue Data Contributor）を付与する。
- `**processing`** に長時間残る Blob は **監視でアラート\*\*。アプリの状態遷移はリポジトリルートの `infra/file-triggered-batch.md` を参照。

## Snowflake 連携（責任境界の要約）

- Terraform で管理するのは **Azure 上の Blob・Queue・Event Grid 等まで**。
- Snowflake 側の Private Link・notification integration・ストレージ連携オブジェクトは **本リポジトリ外**の手順と連携チームの責務。ブループリントの「管理範囲と連携想定」を参照。
