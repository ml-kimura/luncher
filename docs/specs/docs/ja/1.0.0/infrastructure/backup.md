# バックアップ・DR

初期目標とバックアップ対象の方針。出典は `infra/blueprint.md` の「バックアップ・DR」。

## 目標（初期）

| 環境   | RPO    | RTO              |
| ------ | ------ | ---------------- |
| `prod` | 15分   | 4時間            |
| `stg`  | 24時間 | 1営業日          |
| `dev`  | —      | ベストエフォート |

**HA なし構成では RTO は PITR リストア時間に依存**するため、4時間目標は**リストア演習で実測**し、達成できない場合は HA 導入または目標の見直しを判断する。

## コンポーネント別

- **PostgreSQL**: 自動バックアップと PITR を有効化。**リストア演習で実測 RTO**を記録する。
- **Blob Storage**: **コンテナソフトデリートおよび Blob ソフトデリート**を有効化。保持期間とバージョニングはコンテナごとに設定（[ストレージ](./storage.md) の表）。
- **Container Registry**: `prod` でイメージの誤削除防止（タグロック、保持ポリシー）。Dockerfile は Git にあるため **geo-replication は不要**（障害時は再ビルドで復旧）。
- **重要データ**: ペアリージョン冗長などを検討し、フェイルオーバーは計画して検証する。
